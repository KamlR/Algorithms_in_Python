## Объяснение работы алгоритма
### Вступление
У нас есть наиваный алгоритм, который работает за O(n*m). Есть алгоритм, рассмотренный в task1, который работает быстрее наивного, но сильно завязан на хэш функции.
Ключевая идея алгоритма КМП заключается в использовании информации о структуре самого шаблона для предотвращения повторных сравнений символов текста.
При возникновении несовпадения символов алгоритм не возвращается назад по тексту, а сдвигает шаблон таким образом, чтобы уже найденные совпадения были использованы повторно.
Для этого заранее вычисляется вспомогательный массив LPS (Longest Proper Prefix which is also Suffix).

### Структура
Сам алгоритм состоит из двух частей. 
1. Анализ самого паттерна
2. Поиск вхождений паттерна в нашу строку с использованием информации с шага 1.

### Разбор 1 шага
Массив LPS строится для шаблона P и определяется следующим образом:
LPS[i] — длина наибольшего собственного префикса строки P[0..i], который одновременно является её суффиксом.
Под собственным префиксом понимается префикс, не равный всей строке целиком.
P = "ababd"
LPS = [0, 0, 1, 2, 0].
Построение работает строго за O(m).

### Разбор шага 2
На втором шаге алгоритма Кнута–Морриса–Пратта выполняется непосредственный поиск шаблона в тексте.
Массив LPS используется для определения оптимального сдвига шаблона в случае несовпадения символов.
При сравнении шаблона P с текстом T может возникнуть ситуация, когда часть символов уже совпала, но очередной символ не совпадает. Наивный алгоритм в этом случае начинает сравнение заново, что приводит к повторной проверке одних и тех же символов. Массив LPS позволяет определить, какая часть уже совпавшего фрагмента шаблона может быть использована повторно.


## Оценка сложности
Построение LPS работает за O(m), где m - длина паттерна. 
Мы проходим индексом i по каждому символу, не смещаясь назад.

Поиск шаблона в тексте за O(n). Указатель i по тексту никогда не уменьшается.
Указатель j по шаблону может уменьшаться, но только за счёт значений LPS.

Суммарно получаем O(n+m), фиксированная сложность.
